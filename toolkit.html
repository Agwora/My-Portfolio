The Toolkit

Tools are just instruments. What matters is knowing which one to reach for — and when to build something new. Here's what I currently reach for most often, organized by how I think about problems.

📊 Statistics & Mathematical Modeling
  • Hypothesis testing (Z, t, chi-square, two-proportion)
  • Linear & logistic regression
  • Extreme Value Theory (GEV, Block Maxima)
  • Monte Carlo methods & simulation
  • Stochastic processes (exponential decay, logistic growth)
  • Bayesian inference & confidence intervals
  • A/B testing frameworks
🤖 Machine Learning & Deep Learning
  • XGBoost, Random Forest, Scikit-learn
  • PyTorch (neural networks, custom architectures)
  • LSTM (time-series forecasting)
  • Prophet (seasonal decomposition, forecasting)
  • ResNet50 (transfer learning, satellite imagery)
  • Collaborative filtering (recommendation systems)
  • Classification & regression (binary, multi-class)
⚙️ MLOps & Production Systems
  • FastAPI (REST APIs, async, batch prediction)
  • Docker & docker-compose (containerization)
  • MLflow (experiment tracking, model registry)
  • DVC (data versioning, S3-compatible storage)
  • PSI drift detection & automated retraining alerts
  • SMOTE (class imbalance handling)
  • ROC threshold optimization (business ROI alignment)
  • Rule-based fallbacks (fail-safe design)
  • Unit testing (85%+ coverage with pytest)
🧠 LLMs, RAG & Programming
  • LangChain (RAG pipeline orchestration)

  • Python (pandas, numpy, sklearn, PyTorch)
  • SQL (complex joins, window functions, aggregations)
  • YouTube API, web scraping (BeautifulSoup)
  • Flask (lightweight web apps)
  • Git, Jupyter, VS Code
🧠 The invisible toolkit — how I think
Asking "why" five times deep
Explaining stats & ML to non-technical stakeholders
Structuring ambiguous, messy problems
Finding the question behind the question
Failing fast, iterating faster
Translating uncertainty into actionable insights

"Data is never neutral. Every dataset carries the fingerprints of how it was collected, who asked the question, and what was left out. My job is not to find answers. My job is to find better questions. A p-value is not a conclusion. A dashboard is not a decision. I am here to translate between the certain language of numbers and the uncertain reality of humans."
